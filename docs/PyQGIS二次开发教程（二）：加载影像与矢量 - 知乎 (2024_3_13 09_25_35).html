二 绑定图层树与地图画布

我们在上一篇教程中分析了ArcMap 10.7的经典界面设计：

其中最重要的部件为图层树和地图画布，软件使用者首先会用不同方式将遥感影像或矢量拖入图层树中，此时，地图画布将会根据图层树的前后顺序渲染遥感影像和矢量并显示在地图画布中。我们在加载遥感影像和矢量文件前，需要先实现图层树与地图画布的双向绑定。

我们之前提到了MainWindow的ui对象树：

其中的frame将会用来放置地图画布，dockWidgetContents将会用来放置图层树。我们打开根目录的mainWindow.py，并输入以下代码：

from qgis.PyQt.QtWidgets import QMainWindow
from qgis.core import QgsProject,QgsLayerTreeModel
from qgis.gui import QgsLayerTreeView,QgsMapCanvas,QgsLayerTreeMapCanvasBridge
from ui.myWindow import Ui_MainWindow
from PyQt5.QtWidgets import QVBoxLayout,QHBoxLayout
PROJECT = QgsProject.instance()
class MainWindow(QMainWindow, Ui_MainWindow):
    def __init__(self):
        super(MainWindow, self).__init__()
        self.setupUi(self)
        # 1 修改标题
        self.setWindowTitle("QGIS自定义界面")
        # 2 初始化图层树
        vl = QVBoxLayout(self.dockWidgetContents)
        self.layerTreeView = QgsLayerTreeView(self)
        vl.addWidget(self.layerTreeView)
        # 3 初始化地图画布
        self.mapCanvas = QgsMapCanvas(self)
        hl = QHBoxLayout(self.frame)
        hl.setContentsMargins(0,0,0,0) #设置周围间距
        hl.addWidget(self.mapCanvas)
        # 4 设置图层树风格
        self.model = QgsLayerTreeModel(PROJECT.layerTreeRoot(),self)
        self.model.setFlag(QgsLayerTreeModel.AllowNodeRename) #允许图层节点重命名
        self.model.setFlag(QgsLayerTreeModel.AllowNodeReorder) #允许图层拖拽排序
        self.model.setFlag(QgsLayerTreeModel.AllowNodeChangeVisibility) #允许改变图层节点可视性
        self.model.setFlag(QgsLayerTreeModel.ShowLegendAsTree) #展示图例
        self.model.setAutoCollapseLegendNodes(10) #当节点数大于等于10时自动折叠
        self.layerTreeView.setModel(self.model)
        # 4 建立图层树与地图画布的桥接
        self.layerTreeBridge = QgsLayerTreeMapCanvasBridge(PROJECT.layerTreeRoot(),self.mapCanvas,self)

首先，我们需要了解pyqt的Layout知识点，GUI的布局管理是GUI开发中非常重要的一个环节，一个设计良好的GUI界面，其对应的布局排布也是非常必须的。其中布局管理就是要管理窗口中部件的放置。常用到的布局类有QHBoxLayout,QVBoxLayout,QGridLayout QFormLayout4种，分别水平排列布局，垂直排列布局,网格布局管理器,以及表格排列布局:

  • 水平布局管理器（QHBoxLayout）可以把添加的控件以水平的顺序依次排开；
  • 垂直布局管理器（QVBoxLayout）可以把添加的控件以垂直的顺序依次排开；
  • 网格布局管理器（QGridLayout）可以以网格的形式，把添加的控件以一定矩阵排列；
  • 窗体布局管理器（QFormLayout）可以以两列的形式排列所添加的控件

很明显，图层树需要以自上到下的顺序依次排开，所以图层树使用了QVBoxLayout垂直布局，而地图画布目前只有一个frame，不管设置成什么布局都不影响，我们先设为QHBoxLayout水平布局，以后如果设计到多地图画布的时候再详细设计。

我们设置好图层树风格和图层树与地图画布的桥接后，可以尝试着运行以下代码了。我们运行main.py，弹出了如下的窗口:

三 图层（QgsLayer）

加载进软件的矢量、栅格等等都被视为一个个的“图层对象”，被我们之前创建的图层树（QgsLayerTree）管理起来，我们分别按栅格与矢量图层分别讲解。

3.1 尝试使用栅格图层（QgsRasterLayer）

为了方便代码的管理，我们不能在mainWindow里写太多其他的代码，我们在Pycharm中右键>new>PythonPackage新建一个python文件夹，名为qgisUtils，然后我们在qgisUtils中新建一个名为qgisLayerUtils的py文件，现在我们的文件树应该如下所示：

我们在qgisLayerUtils中写入如下代码：

from qgis.core import QgsMapLayer,QgsRasterLayer,QgsVectorLayer

​

使用QgsRasterLayer可以很方便的就创建好软件需要的QgsMapLayer图层对象，访问栅格文件，用到了GDAL库。它支持多种文件格式。如果你在打开某些文件时遇到问题，请检查你的GDAL是否支持（默认情况下并非所有格式都可用）。我们可以尝试获取一些rasterLayer的属性看看，在获取rasterLayer的属性前，我们需要去搜寻至少一张栅格影像（.tif）和一套矢量文件（.shp）：
