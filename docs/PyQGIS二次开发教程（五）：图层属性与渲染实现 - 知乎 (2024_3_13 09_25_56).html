二 栅格图层与矢量图层基本信息获取

2.1 栅格图层

我们从之前的教程已经了解了栅格图层的对象为：

QgsRasterLayer （ qgis.core.QgsRasterLayer ）

我们在第一章教程 D:\learnPyQGIS\qgisUtils\qgisLayerUtils.py 中曾经写过简单的栅格图层基本获取方法函数：

def getRasterLayerAttrs(rasterLayer:QgsRasterLayer):
    print("name: ", rasterLayer.name()) # 图层名
    print("type: ", rasterLayer.type()) # 栅格还是矢量图层
    print("height - width: ", rasterLayer.height(),rasterLayer.width()) #尺寸
    print("bands: ", rasterLayer.bandCount()) #波段数
    print("extent", rasterLayer.extent()) #外接矩形范围
    print("source", rasterLayer.source()) #图层的源文件地址
    print("crs", rasterLayer.crs())  # 图层的坐标系统

我们在这一章需要将这个函数改写，可以返回一个字典，提供栅格图层的一些基本信息，我们在栅格属性页面图中可以看到我们需要的信息如下：

  • 栅格图层名称
  • 栅格图层源路径
  • 栅格图层文件占用内存大小
  • 栅格范围
  • 栅格宽度、高度
  • 栅格数据类型
  • 栅格波段数量
  • 栅格坐标系

2.1.1 重构getRasterLayerAttrs函数

我们的新函数为如下：

def getRasterLayerAttrs(rasterLayer:QgsRasterLayer):
​
    rdp : QgsRasterDataProvider = rasterLayer.dataProvider()
    crs : QgsCoordinateReferenceSystem = rasterLayer.crs()
    extent: QgsRectangle = rasterLayer.extent()
    resDict = {
        "name" : rasterLayer.name(),
        "source" : rasterLayer.source(),
        "memory" : getFileSize(rasterLayer.source()),
        "extent" : f"min:[{extent.xMinimum():.6f},{extent.yMinimum():.6f}]; max:[{extent.xMaximum():.6f},{extent.yMaximum():.6f}]",
        "width" : f"{rasterLayer.width()}",
        "height" : f"{rasterLayer.height()}",
        "dataType" : qgisDataTypeDict[rdp.dataType(1)],
        "bands" : f"{rasterLayer.bandCount()}",
        "crs" : crs.description()
    }
    return resDict

从函数中可以看到，我们出现了三个新的Qgis类：QgsRasterDataProvider、QgsCoordinateReferenceSystem、QgsRectangle：

QgsRasterDataProvider（from qgis.core import QgsRasterDataProvider）是栅格图层的数据提供库，使用rasterLayer.dataProvider()便可获得该对象，可以通过该对象获得一些栅格数据源信息，例如：rdp.dataType(1) 代表栅格图层第一个波段的数据类型，这个函数会返回一个int类型，通过查阅qgis的api，我们可以得到下面的qgis数据类型字典映射：

qgisDataTypeDict = {
    0 : "UnknownDataType",
    1 : "Uint8",
    2 : "UInt16",
    3 : "Int16",
    4 : "UInt32",
    5 : "Int32",
    6 : "Float32",
    7 : "Float64",
    8 : "CInt16",
    9 : "CInt32",
    10 : "CFloat32",
    11 : "CFloat64",
    12 : "ARGB32",
    13 : "ARGB32_Premultiplied"
}

QgsCoordinateReferenceSystem（from qgis.core import QgsCoordinateReferenceSystem）是qgis坐标系统对象，使用rasterLayer.crs()便可获得该对象，可以通过该对象获得图层的坐标系统信息，我们这里只简单使用其中的文本描述，该坐标系统在后续章节还会时常使用： crs.description()

QgsRectangle（from qgis.core import QgsRectangle）是Qgis的一个代表矩形的对象，我们通过rasterLayer.extent()可以获得该栅格的一个最小外接矩形范围，从该对象可以获得栅格的范围：f"min:[{extent.xMinimum():.6f},{extent.yMinimum():.6f}]; max:[{extent.xMaximum():.6f},{extent.yMaximum():.6f}]"

2.1.2 获取文件大小

在上面函数可以看到一个memory字典中获取方法是 getFileSize(rasterLayer.source()) ，这是一个获取文件大小的函数，我们在qgisUtils中创建新函数： D:\learnPyQGIS\qgisUtils\yoyiFile.py

import os
import os.path as osp
​
def getFileSize(filePath):
    fsize = osp.getsize(filePath)  # 返回的是字节大小
​
    if fsize < 1024:
        return f"{round(fsize, 2)}Byte"
    else:
        KBX = fsize / 1024
        if KBX < 1024:
            return f"{round(KBX, 2)}Kb"
        else:
            MBX = KBX / 1024
            if MBX < 1024:
                return f"{round(MBX, 2)}Mb"
            else:
                return f"{round(MBX/1024,2)}Gb"

2.2.1 重构getVectorLayerAttrs函数

我们的新函数为如下：

def getVectorLayerAttrs(vectorLayer:QgsVectorLayer):
    vdp : QgsVectorDataProvider = vectorLayer.dataProvider()
    crs: QgsCoordinateReferenceSystem = vectorLayer.crs()
    extent: QgsRectangle = vectorLayer.extent()
    resDict = {
        "name" : vectorLayer.name(),
        "source" : vectorLayer.source(),
        "memory": getFileSize(vectorLayer.source()),
        "extent" : f"min:[{extent.xMinimum():.6f},{extent.yMinimum():.6f}]; max:[{extent.xMaximum():.6f},{extent.yMaximum():.6f}]",
        "geoType" : QgsWkbTypes.geometryDisplayString(vectorLayer.geometryType()),
        "featureNum" : f"{vectorLayer.featureCount()}",
        "encoding" : vdp.encoding(),
        "crs" : crs.description(),
        "dpSource" : vdp.description()
    }
    return resDict

上面的对象基本与栅格图层一致，除了 vdp : QgsVectorDataProvider = vectorLayer.dataProvider() ，编码可以由vdp.encoding()获得，数据源可以由vdp.description()获得，几何图形类型可以由 QgsWkbTypes .geometryDisplayString(vectorLayer.geometryType()),获得。

3.1 了解QListWidget

我们翻阅QT的官方文档：QListWidget is a convenience class that provides a list view similar to the one supplied by QListView, but with a classic item-based interface for adding and removing items. QListWidget uses an internal model to manage each QListWidgetItem in the list.QListWidget 是一个继承自 QListView 的类，其实就是 QListView 的一个很经典的 列表 交互控件，在QListWidget 里面提供了非常方便的基于 每一个 QListWidgetItem 的 管理 类，从添加、选择、删除 列表子Item到 QListWidget 里，都是封装的非常完善的，我们可以用QListWidget作为图层户型左边的索引列表widget实现。

3.2 了解QTabWidget

我们翻阅QT的官方文档：

A tab widget provides a tab bar (see QTabBar ) and a “page area” that is used to display pages related to each tab. By default, the tab bar is shown above the page area, but different configurations are available (see TabPosition ). Each tab is associated with a different widget (called a page). Only the current page is shown in the page area; all the other pages are hidden. The user can show a different page by clicking on its tab or by pressing its Alt+letter shortcut if it has one.

QTabWidget提供一个“选项卡栏”和一个“页面区域”，用于显示与每个选项卡相关的页面。默认情况下，选项卡栏显示在页面区域上方，但可以设置TabPosition。每个选项卡都与不同的Widget（称为页面）相关联。页面区域只显示当前页面；所有其他页面都被隐藏。
